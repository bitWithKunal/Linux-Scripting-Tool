AWK

AWK

AWK Overview

Description

AWK is a powerful text-processing tool. It reads files line by line, splits each line into fields, and lets you apply patterns and actions. Common in log analysis, CSV processing, and reporting.

Use Cases

  • Summarizing CSV data
  • Log analysis
  • Quick filtering

1. Hello World

awk 'BEGIN {print "Hello, World!"}'

üëâ BEGIN runs before any line is read.

2. Sum Column 2

awk '{sum += $2} END {print sum}' data.txt

üëâ $2 is the 2nd field. END runs after all lines.

3. Print Line Numbers

awk '{print NR, $0}' data.txt

üëâ NR = line number, $0 = full line.

4. Print First Column

awk '{print $1}' data.txt

üëâ $1 = first field in the line.

5. Average of Column 3

awk '{sum+=$3} END {print sum/NR}' data.txt

üëâ NR counts lines. $3 = 3rd field.

6. Filter "ERROR" Lines

awk '/ERROR/ {print}' log.txt

üëâ Pattern match. Prints only matching lines.

7. FNR vs NR

awk '{print FNR, NR, $0}' file1 file2

üëâ FNR = line # in current file. NR = global line #.

8. Field Separator (CSV)

awk -F, '{print $1, $3}' data.csv

üëâ -F, sets delimiter to comma.

9. Conditional Filtering

awk '$3 > 50 {print $1, $3}' data.txt

üëâ Only prints if 3rd field is > 50.

10. Range of Lines

awk 'NR==2,NR==5 {print}' data.txt

üëâ Prints lines 2 through 5.

11. Pretty Printing

awk '{printf "%-10s %5d\n", $1, $2}' data.txt

üëâ printf lets you format output neatly.

12. String Functions

awk '{print length($1), substr($1,1,3)}' data.txt

üëâ length(), substr() work on fields.

13. Arrays & Counting

awk '{count[$1]++} END {for (c in count) print c, count[c]}' data.txt

üëâ Uses associative arrays to count values.

14. Redirect Output

awk '{print $1}' data.txt > names.txt

üëâ Redirects AWK output into a new file.

15. Multiple Files

awk 'FNR==1 {print "==>",FILENAME} {print}' file1 file2

üëâ FNR restarts per file. Prints filename headers.

grep is a command-line tool for searching text using regular expressions. It is extremely fast and widely used for log scanning, pattern matching, and filtering output.

  • Pattern matching

üëâ Finds all lines containing "ERROR".

2. Case-Insensitive Search

üëâ Ignores case (matches ERROR, Error, error).

3. Count Matches

üëâ Prints the number of matching lines.

4. Show Line Numbers

grep -n "ERROR" logfile.txt

üëâ Shows matching line numbers.

5. Invert Match

grep -v "INFO" logfile.txt

üëâ Shows lines that do not contain "INFO".

6. Match Whole Word

grep -w "root" /etc/passwd

üëâ Matches "root" but not "roots" or "groot".

7. Recursive Search

grep -r "TODO" ./project

üëâ Searches inside all files under ./project.

8. Only Filenames

grep -l "main()" *.c

üëâ Prints only filenames that matched.

9. Exclude Filenames

grep -L "TODO" *.txt

üëâ Prints filenames with no match.

10. Extended Regex

grep -E "cat|dog" animals.txt

üëâ Uses OR pattern (matches cat or dog).

11. Show Context (Before)

grep -B 2 "ERROR" logfile.txt

üëâ Shows 2 lines before each match.

12. Show Context (After)

grep -A 3 "ERROR" logfile.txt

üëâ Shows 3 lines after each match.

13. Full Context

grep -C 2 "ERROR" logfile.txt

üëâ Shows 2 lines before and after matches.

14. Match Start of Line

grep "^root" /etc/passwd

üëâ Matches lines that start with "root".

15. Match End of Line

grep "bash$" /etc/passwd

üëâ Matches lines that end with "bash".

Log Monitor + Alert

Project Overview

This Bash project monitors application logs and raises an alert if the number of ERROR entries crosses a threshold. It‚Äôs a simple but powerful script for DevOps engineers and system admins.

Real-World Use Cases

  • Monitor server logs for repeated errors
  • Detect application crashes early
  • Alert on suspicious activities in logs
  • Integrate with Slack/Email notifications

Implementation Script 

#!/bin/bash
# ==========================
# Log Monitor + Alert Script
# ==========================

# 1. Set the log file path
logfile="app.log"

# 2. Define the error threshold
threshold=5

# 3. Count occurrences of the word "ERROR" in the log file
#    - grep -c : count matches
#    - "$logfile": file to scan
count=$(grep -c "ERROR" "$logfile")

# 4. Compare the count with threshold
if (( count > threshold )); then
    echo "üö® High error rate detected! ($count errors)"
    # Optional: send alert via email or Slack
    # mail -s "High Error Alert" admin@example.com <<< "Found $count errors in $logfile"
else
    echo "‚úÖ All good, only $count errors."
fi

  3. Run it: ./log_monitor.sh
  4. Modify threshold as needed
  5. Extend with Slack/Email alerts for production use

STA Constraint Checker

Project Overview

This TCL project parses an SDC (Synopsys Design Constraints) file and checks whether all critical constraints are defined. It helps Physical Design engineers quickly identify missing create_clock, set_input_delay, and set_output_delay commands.

Real-World Use Cases

  • Verify timing constraints in STA setup
  • Ensure complete SDC files before sign-off
  • Catch missing delays that can cause tape-out issues
  • Validate clock definitions early in flow

Implementation Script 

# ===============================
# STA Constraint Checker in TCL
# ===============================

# 1. Open the SDC file for reading
set fh [open "constraints.sdc" r]

# 2. Read the file contents into a variable
set data [read $fh]

# 3. Close the file after reading
close $fh

# 4. Check for required constraints
#    - create_clock : mandatory for defining clocks
#    - set_input_delay : timing info for input ports
#    - set_output_delay : timing info for output ports

if {![regexp {create_clock} $data]} {
    puts "‚ö†Ô∏è Missing clock definition!"
}
if {![regexp {set_input_delay} $data]} {
    puts "‚ö†Ô∏è Missing input delay!"
}
if {![regexp {set_output_delay} $data]} {
    puts "‚ö†Ô∏è Missing output delay!"
}

# End of Script

How to Run

  1. Save the script as sta_checker.tcl
  2. Ensure you have an constraints.sdc file in the same directory
  3. Run in terminal: tclsh sta_checker.tcl
  4. Check the output for warnings ‚ö†Ô∏è about missing constraints
  5. Extend it to check more SDC commands (like set_clock_uncertainty)
SED Project

Config Cleanup Automation

Project Overview

This project uses Sed (Stream Editor) to clean up messy configuration files. It automatically removes comments (# ...), trims extra spaces, deletes empty lines, and saves a cleaner version. It‚Äôs ideal for DevOps automation, system setup scripts, and config migrations.

Real-World Use Cases

  • Clean config files before deployment
  • Prepare sanitized config for version control
  • Automate formatting for multiple .ini files
  • Speed up manual review during system audits

Implementation Script 

#!/bin/bash
# ================================
# Config Cleanup Automation (SED)
# ================================

# Input configuration file
input="config.ini"

# Output cleaned configuration file
output="cleaned_config.ini"

# Sed command explanation:
#   1. s/#.*//              ‚Üí Remove everything after "#" (comments)
#   2. s/^[[:space:]]+//    ‚Üí Trim leading spaces
#   3. s/[[:space:]]+$//    ‚Üí Trim trailing spaces
#   4. /^$/d                ‚Üí Delete empty lines
sed -E 's/#.*//; s/^[[:space:]]+//; s/[[:space:]]+$//; /^$/d' "$input" > "$output"

echo "‚úÖ Config cleanup complete: $output"

How to Run

  1. Save the script as cleanup.sh
  2. Make it executable: chmod +x cleanup.sh
  3. Ensure config.ini is in the same directory
  4. Run the script: ./cleanup.sh
  5. Check the cleaned version in cleaned_config.ini
AWK Project

CSV Summarizer

Project Overview

This project uses Awk to process CSV files and compute the average of column 2 grouped by column 1. It‚Äôs a handy tool for data analysis, log processing, and quick reporting without heavy tools like Excel or Python.

Real-World Use Cases

  • Summarize sales data by region
  • Compute averages in CSV log files
  • Group and analyze sensor readings
  • Generate quick reports from user data

Implementation Script 

#!/usr/bin/awk -f
# ===================================
# CSV Summarizer (Average Calculator)
# ===================================

# Use comma as field separator (-F,)
# $1 ‚Üí first column (group key)
# $2 ‚Üí second column (numeric values)

# BEGIN block: can initialize if needed
# Process block: sum values and count occurrences per key
{
    sum[$1] += $2
    count[$1]++
}

# END block: compute and print averages
END {
    for (key in sum) {
        avg = sum[key] / count[key]
        printf "%s ‚Üí Average: %.2f\n", key, avg
    }
}

How to Run

  1. Prepare a CSV file, e.g. data.csv: 
    Region1,100
Region1,200
Region2,150
Region2,250
  2. Run the script directly: awk -F, -f csv_summarizer.awk data.csv
  3. Or use a one-liner: awk -F, '{sum[$1]+=$2; count[$1]++} END {for (k in sum) printf "%s ‚Üí Avg: %.2f\n", k, sum[k]/count[k]}' data.csv
  4. Output will show each group (Column 1) and its average of Column 2.
